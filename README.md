##1.通用原则
>作为成功的项目的一员，很重要的一点是意识到只为自己写代码是很糟糕的行为。如果将有成千上万的人使用你的代码
>那么你需要编写最具明确性的代码，而不是以自我的喜好来彰显自己的智商。 - Idan Gazit

* 别想着过早的优化代码。先得保证他们可读又可理解才行
* 在任何代码库中，无论有多少人参与贡献，所有代码都应该如同一个人编写的一样
* 严格执行一致认可的风格

##2.编码
统一为UTF-8

##3.格式
* 统一使用4个空格缩进
* *永远不要*在缩进时混用空格和制表符(tab)
